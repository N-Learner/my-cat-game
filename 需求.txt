# 互动玩法与细节
- 名称: 猫咪叠叠乐 (Cat Stacker)
- 玩法类型: {gameplay_details:

  【定位与核心差异】
  合成驱动的物理堆叠游戏，但与"合成大西瓜"类游戏有本质区别：
  - 不是封闭容器填充，而是在一个没有围栏的开放平台上堆高
  - 不是垂直掉落，而是弹弓抛射（控制角度+力度）
  - 物体不是被动死物，猫咪会自主行为（扭动、伸懒腰、张望）
  - 拥有猫爪抓握系统，猫咪着陆后会"抓住"表面，受冲击后松手滑落
  - 核心目标是"堆得最高"而非"合成最大"
  - 定期触发灾难事件（地震、猫咪打喷嚏、猫咪打架），制造连锁翻车

  【核心循环】
  弹弓瞄准抛出猫咪 → 猫咪飞行落上平台/堆叠体 → 猫爪抓握短暂稳定 → 物理碰撞与平衡博弈 → 同级猫咪接触合成升级 → 灾难事件打破平衡 → 继续抛射，追求最大高度与高分

  【场景结构 — 开放式平台】
  - 屏幕中下方（Y轴82%位置）有一个无围栏的水平平台
  - 平台宽度为屏幕宽度的55%，居中放置
  - 平台两端以红色圆点标注边缘，提示玩家"这里没有墙"
  - 猫咪重心超出平台边缘30%即脱离平台判定，会自由坠落
  - 猫咪坠落屏幕下方100px或飞出屏幕左右100px判定为"丢失"
  - 平台造型为梯形立体感（顶宽>底宽），线性渐变，有顶部高光描边

  【投放机制 — 弹弓抛射】
  - 玩家在屏幕下方发射区按住拖动形成弹弓拉伸，松手释放抛出猫咪
  - 拖动方向决定抛射角度，拖动距离决定力度
  - 最大拉伸距离为屏幕宽度的35%，超出不增加力度
  - 初始速度范围：最小250px/s，最大800px/s，线性映射拉伸距离
  - 拖动过程中显示抛物线轨迹预览（15个点，每点间隔0.06s，透明度递减）
  - 底部显示力度指示条（绿→黄→红）
  - 抛出时附加随机角速度±1 rad/s
  - 每次投放后0.6s冷却

  【猫咪等级体系 — 8级】
  Lv1 小橘团: r=14, round, mass=0.8, color=#FFB347, fric=0.35, rest=0.30
  Lv2 蓝短:   r=17, round, mass=1.2, color=#87CEEB, fric=0.30, rest=0.32
  Lv3 布偶:   r=21, oval(hw=r×1.2,hh=r×0.85), mass=1.8, color=#DDA0DD, fric=0.42, rest=0.22
  Lv4 英短:   r=25, chubby(hw=r×1.1,hh=r×0.95), mass=2.5, color=#98FB98, fric=0.48, rest=0.20
  Lv5 金渐层: r=30, round, mass=3.5, color=#F4D03F, fric=0.38, rest=0.25
  Lv6 黑猫:   r=35, oval, mass=5.0, color=#2d3436, eye=#FFD700, fric=0.44, rest=0.18
  Lv7 大橘:   r=41, chubby, mass=7.0, color=#E17055, fric=0.50, rest=0.12
  Lv8 猫王:   r=48, round, mass=10.0, color=#FFD700, eye=#C0392B, fric=0.55, rest=0.08
  
  玩家投放时仅生成Lv1~Lv5，权重分别为40%/28%/16%/9%/5%。
  "下一个"预览显示在发射区右侧。

  【合成机制】
  - 两只同等级猫咪碰撞并保持接触≥0.12s后合成为高一级
  - 合成在两者中点生成新猫咪，继承双方速度的25%，附加向上弹起-100px/s
  - Lv8为最高级不再合成
  - 合成时触发：爆发粒子(14+level×4颗) + 圆环冲击波(12颗白色粒子) + 屏幕闪光(0.12透明度) + 相机震动
  - 新猫咪显示happy表情1s，发光效果0.6s

  【猫爪抓握系统 — 核心差异特性】
  - 猫咪着陆时获得1.5s的抓握时间
  - 抓握力350单位，作用为速度阻尼（对水平速度和角速度施加摩擦）
  - 抓握时间每秒自然衰减70%（指数递减），约2~3s后趋近于零
  - 碰撞冲量>60时抓握时间×0.7，地震时所有猫咪抓握×0.3
  - 视觉表现：猫咪脚下两个黄色小圆点（爪印），透明度随抓握力衰减
  - 设计意图：创造"猫咪死死抓住→慢慢滑落→终于掉下去"的戏剧弧线

  【猫咪自主行为系统 — 核心差异特性】
  每只猫咪独立行为计时器，基础间隔3s×(0.5~1.5随机)，仅grounded状态触发：
  - 伸懒腰(25%): 0.8s, squash目标值1.15，猫咪纵向拉伸
  - 扭动(20%): 0.5s, 水平速度±20px/s轻微移位，15%概率触发meow音效
  - 张望(10%): 1.0s, 附加角速度±0.15 rad/s
  - 发呆(45%): 无动作

  【灾难事件系统 — 核心差异特性】

  地震:
  - 间隔15~28s随机，持续0.5s
  - 所有着地猫咪受随机水平力
  - 附带：所有猫抓握×0.3、相机震动4、屏幕橙红滤镜
  - UI提示"🌍 地震!"
  - 音效：低频噪声0.25s

  猫咪打喷嚏:
  - 间隔8~20s随机
  - 随机一只着地猫咪向上弹起120~200px/s，水平±50px/s，抓握清零
  - 表情surprised 0.8s，头顶4颗粒子
  - UI提示"🤧 {猫咪名称}打了个喷嚏!"
  - 音效：sine 800→200Hz

  猫咪打架:
  - 间隔12~25s随机
  - 距离最近的两只着地猫咪沿连线互推80~140px/s，各上弹30px/s，抓握×0.5
  - 双方surprised表情0.8s
  - UI提示"😾 猫咪打架了!"
  - 音效：meow

  【计分方式】
  - 投放着陆: +10
  - 合成得分: 50 × (新等级+1)
  - 连击倍率: 合成得分 × (1 + 连击次数×0.5)，2.0s窗口内连续合成视为连击
  - 高度成就: 结算时展示最高高度（像素÷30=米）

  【失败条件 — 危险条机制】
  - 危险条0~100，右上角竖向进度条显示
  - 自然衰减: 每秒-2.5
  - 猫咪坠落屏幕外: +18
  - 猫咪飞出侧边: +10.8
  - 堆叠不稳定: 着陆猫速度>80px/s时按速度比例持续累加
  - 猫咪超50只强制裁剪: 每移除一只+10
  - 危险值≥100触发Game Over
  - 视觉: 65%以下橙色渐变，65%以上红色渐变

  【相机系统】
  - 垂直跟随最高猫咪，保持屏幕上方40%空间
  - camY以lerp系数3/s平滑插值
  - 平台左侧显示高度标尺，每60px(2m)一格
  - 发射区不受相机偏移影响，锚定屏幕底部

  【肉鸽Buff系统】
  每投放10次触发一次三选一（从4个中随机3个），暂停游戏选择，整局叠加生效：
  - 🧲 磁力爪: 猫咪受同级猫吸引（半径cr×4+15×层数，力25×层数）
  - 🪶 轻飘飘: 重力降低（1-0.2×层数，上限3层=60%重力）
  - 🧴 加固胶水: 摩擦力倍率提升（1+0.5×层数）
  - 🐾 铁爪功: 抓握力倍率提升（1+0.4×层数）
  选中的Buff以小标签形式显示在屏幕顶部居中

  【结算展示】
  Game Over后展示：
  - 本局得分（大号高亮）、历史最高分
  - 统计：投放数、合成次数、最大连锁、最高高度(m)、最高等级+名称
  - 成就徽章（满足条件时显示）:
    - 最大连锁≥3: "🏆 连锁大师 ×{n}"
    - 最高等级≥6: "👑 合成达人 Lv{n}"
    - 最高高度≥5m: "🗼 摩天大楼 {n}m"
    - 投放数≥50: "🐱 猫咪收集家"
  - 一键"再来一局"按钮

  【模式】
  无尽模式，目标刷高分与最大高度。
}

# 产物技术与合规约束（严格遵守）
{tech_constraints:
- 交付形态: 单文件。所有CSS和JavaScript代码直接内联到一个index.html中。Canvas绘制所有游戏图形，UI元素使用DOM覆盖层（pointer-events:none + 局部pointer-events:auto）。
- 纯离线运行: 整个作品必须完全离线运行，所有资源包含在html文件内。
- 无网络请求: 绝对禁止使用任何形式的网络请求API，包括fetch, XMLHttpRequest, axios, WebSocket等。
- 无外部资源: 禁止从任何外部CDN或服务器加载资源。所有视觉元素用Canvas API绘制。音效使用Web Audio API合成（OscillatorNode/AudioBuffer生成），不加载外部音频文件。不使用外部字体，仅使用system-ui系统字体栈。
- 无外部跳转: 绝对禁止使用<a>标签外链、window.location或任何其他方式跳转到外部链接。
- 轻量化: 最终产物打包为zip后，总体积必须小于8MB（纯代码单文件远小于此限制）。
- 数据存储: localStorage存储最高分（key: cs_hi）和音效开关（key: cs_snd, 值"1"/"0"）。仅在Game Over时写入最高分。
- UI适配:
  - 屏幕方向: 竖屏。
  - 自适应布局: Canvas按devicePixelRatio做高清渲染（dpr上限3），CSS宽高100%。页面无横向滚动条。resize事件触发重新计算所有布局参数（平台位置/宽度、发射区位置等）。
  - 触控: -webkit-tap-highlight-color:transparent, -webkit-touch-callout:none, user-select:none, touch-action:none(canvas)。
- 错误处理: 全局try-catch包裹所有游戏代码，出错时在页面上显示友好提示：{error_handling_text: 哎呀，出错了，请重启试试吧~ (附带error.message)}。
- 合规要求: {negative_prompt: 内容不得包含任何违法违规、IP侵权、不利于未成年人的内容。猫咪为原创抽象矢量绘制，不模仿任何已有IP形象。}

- 物理引擎规格（内置实现，不使用外部库）:
  - 重力: 750 px/s²
  - 每帧物理迭代: 10次子步长
  - 速度阻尼: 0.997/子步长，角速度阻尼: 0.993/子步长
  - 碰撞检测: 圆-圆检测（collisionRadius = (hw+hh)/2近似）
  - 碰撞响应: 按质量反比分离(80%) + 法向冲量 + 切向摩擦(0.4系数) + 角速度传递(0.15系数)
  - 场上最大猫咪数: 50只，超出裁剪最早生成的
  - 单帧dt上限: 0.04s防止物理爆炸

- 性能要求:
  - 中端手机≤30只猫时≥50fps
  - 粒子系统自然生命周期回收，无持续增长
  - devicePixelRatio上限3
}

# 游戏状态机
{state_machine:
  [menu] --点击"开始游戏"--> [playing]
  [playing] --点击暂停按钮--> [paused]
  [paused] --点击继续按钮--> [playing]
  [playing] --每10次投放后延迟0.8s--> [buff_select]
  [buff_select] --点击选择一个Buff--> [playing]
  [playing] --危险值≥100--> [gameover]
  [gameover] --点击"再来一局"--> [playing]
  
  仅[playing]状态下响应拖动和投放操作。
  仅[playing]状态下运行物理和游戏逻辑更新。
  所有状态下持续渲染画面（暂停时画面静止但仍绘制）。
}
