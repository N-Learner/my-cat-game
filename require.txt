# 角色
{role: 你是一位资深的 HTML5 Canvas 开发者，精通移动端 Web 内容开发、物理引擎模拟（Matter.js 风格或自定义物理）、图像处理及极致的性能优化。} 

# 核心目标
{goal: 开发一款名为《猫咪叠叠乐：物理灾难秀》的纯离线 HTML5 互动作品。这是一款基于物理堆叠、拥有动态表情反馈、猫咪社交博弈和肉鸽（Roguelite）增益系统的精品小游戏。核心亮点在于：在精美的静态猫咪头像上通过 Canvas 实时重绘动态表情，以及 Game Over 时的慢动作灾难回放。} 

# 互动玩法与细节
- 名称: 猫咪叠叠乐 (Cat Stacker: Physics Disaster Show)
- 玩法类型: {gameplay_details: 
    1. **核心逻辑**：玩家在屏幕顶部左右滑动控制猫咪位置，松手后猫咪受重力下落。目标是在没有围栏的开放平台上堆叠尽可能多的猫咪，追求最高高度。
    2. **物理系统**：
        - **猫爪抓握**：猫咪落地有 1.5s 初始抓握力，受冲击或地震会滑落。
        - **永久形变**：被压在底层的猫咪会随压力和时间逐渐“永久压扁”。
        - **开放平台**：平台随分数推进会缩窄、倾斜甚至左右摇摆。
    3. **动态表情系统（核心特色）**：
        - 使用 6 种猫咪静态图作为底图（橘猫、狸花、布偶、黑猫、灰猫、长毛猫）。
        - 在图片对应的眼睛和嘴巴位置，根据状态覆盖绘制表情：飞行中（恐惧 X X）、高处稳定（得意 ^ v ^）、被压久了（晕眩 @ _ @）、被砸瞬间（痛苦 > <）、边缘挣扎（颤抖 + 冷汗粒子）。
    4. **社交博弈**：
        - 建立猫咪关系矩阵：亲密（吸引+加分）、友好（互动）、不合（排斥+打架扣分）。
        - 抛射前场上相关猫咪头顶显示 ❤️ 或 💢 提示。
    5. **肉鸽 Buff 系统**：
        - 每 10 次投放弹出全屏大卡片三选一。
        - 增益包括：铁爪功（抓握力）、重力微调、磁力吸引、纸箱护栏、九条命（自动复活）等。
        - 屏幕右侧竖排常驻显示 Buff 图标及其叠加层数。
    6. **灾难事件**：随机触发地震、猫咪打喷嚏、集体炸毛等，制造连锁塌方。
    7. **胜利/失败**：无尽模式。堆叠体系彻底崩塌导致“危险条”满格时失败。失败瞬间触发 0.15 倍速慢动作回放，相机拉远，追踪飞得最远的猫。
} 
- 核心交互: {interaction_mode: 顶部水平滑动调整落点，松手投放；点击右侧 Buff 图标查看详情。} 
- 美术风格: {visual_style: 精美 Q 版扁平卡通。背景为深色太空渐变配闪烁星星。UI 采用 Google Material Design 风格，具有毛玻璃质感、圆角卡片和流畅的弹出动画。} 

# 产物技术与合规约束（严格遵守）
{tech_constraints: 
- 交付形态: 单文件 index.html。所有 CSS 和 JS 内联，不引用任何外部资源（如 CDN 库、外部图片、外部字体）。
- 图像处理: 游戏必须预留 6 个隐藏的 `<img>` 标签供我放入 Base64 素材。Canvas 需实现根据猫咪状态在图片上进行 `translate`（发抖）、`scale`（压扁）和 `overlay drawing`（重绘表情）。
- 纯离线运行: 禁止任何 `fetch`, `XMLHttpRequest` 等网络请求。
- 轻量化: 代码逻辑精简，zip 体积需远小于 8MB。
- 数据存储: 使用 `localStorage` 记录历史最高分、最高高度及解锁的图鉴。
- UI 适配: 
  - 屏幕方向: 竖屏。
  - 自适应布局: 必须适配从 iPhone SE 到 iPad 的所有比例，按 `devicePixelRatio` 高清渲染，严禁出现横向滚动条。
- 错误处理: 必须包含 try-catch 块，出错显示：“哎呀，出错了，请重启试试吧~”。
- 合规要求: 内容健康向上，无任何侵权内容。
} 

# 输出结构与验证口径
{output_structure: 
- 输出一个完整的 `index.html`。
- 逻辑需包含：物理引擎、表情状态机、Buff 管理器、社交关系判定、慢动作渲染器、结算卡片生成。
- 验证口径：
    1. 拖动顶部猫咪是否平滑，是否有落点引导虚线。
    2. 猫咪被压时，眼睛是否正确重绘为螺旋眼。
    3. Buff 选择界面是否为全屏大卡片，右侧是否正确叠加层数。
    4. 崩塌时是否确实减速并拉远相机。
    5. 结算时能否生成一张包含“最离谱瞬间”文字的分享卡片截图。
}
